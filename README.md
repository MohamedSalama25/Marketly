# 🏪 Marketly - منصة التجارة الإلكترونية B2B المتكاملة

تطبيق ويب متطور ومتكامل مصمم لربط الموردين والشركات بمتاجر التجزئة، مع نظام ذكي لإدارة العمليات وتتبع سلاسل التوريد وتسهيل التواصل.

---

## 🎯 رؤية المشروع (Vision)
يهدف **Marketly** إلى رقمنة وتسهيل عمليات البيع بالجملة وطلبات التوريد بين الشركات (Suppliers) والمتاجر (Retailers)، مما يقلل من العقبات التقليدية في سلاسل التوريد ويوفر شفافية تامة في تتبع الطلبات والمناديب من خلال واجهات عصرية وأدوات ذكية.

---

## 🚀 لماذا Marketly؟ (المشكلة والحل)

### **المشكلة:**
*   عشوائية التواصل بين الموردين وتجار التجزئة.
*   غياب نظام مركزي لمتابعة المناديب والطلبات بشكل حي.
*   صعوبات في إدارة المرتجعات والشكاوى بشكل يدوي.
*   نقص البيانات والتحليلات اللازمة لاتخاذ قرارات تجارية ذكية.

### **الحل:**
منظومة رقمية موحدة تجمع كافة الأطراف، تعمل على أتمتة دورة حياة الطلب بالكامل، مع توفير أدوات ذكاء اصطناعي للمساعدة (AIAssistant) ونظام محادثة فوري، مما يضمن كفاءة تشغيلية بنسبة 100%.

---

## ✨ المميزات الرئيسية (Key Features)

### � **إدارة الأدوار (Role Management)**
*   **المدير (Admin):** تحكم كامل بالمستخدمين، الصلاحيات، والتقارير المالية.
*   **المورد (Supplier):** إدارة المنتجات، تحديد الأسعار، وتعيين المناديب للطلبات.
*   **المندوب (Delegate):** استلام خطط السير، تحديث حالة التوصيل، وتتبع المواقع.
*   **تاجر التجزئة (Retailer):** الوصول للموردين، طلب الكميات، والاستفادة من العروض.

### 🤖 **مساعد ماركتلي الذكي (Marketly AI Assistant)**
*   مساعد مدمج في اللاندينج بيدج للإجابة على استفسارات العملاء فوراً.
*   يدعم التحدث الصوتي (ElevenLabs API) لتحسين تجربة المستخدم.
*   يوفر معلومات سريعة عن الخدمات، سياسات الشحن، وكيفية التسجيل.

### � **الأدوات والتحليلات**
*   **لوحة تحكم (Dashboard):** رسوم بيانية تفاعلية لمعدلات البيع والنمو.
*   **إدارة المرتجعات:** نظام سلس لمعالجة الطلبات المرتجعة.
*   **تتبع الموقع:** تكامل مع الخرائط لتحديد مواقع المتاجر الجغرافية.

## 📂 الهيكل التنظيمي للمشروع (Project Structure)

يتبع المشروع هيكلاً منظماً يسهل عملية الصيانة والتطوير المستمر:

```bash
src/
├── Redux/            # إدارة الحالة العالمية (Global State) والبيانات
├── Supabase/         # ملفات الربط مع قاعدة البيانات السحابية (DB & Auth)
├── Pages/            # الصفحات الأساسية (Landing, Dashboard, Auth, etc.)
├── Components/       # المكونات المقسمة وظيفياً:
│   ├── assistant/        # المساعد الذكي (AIAssistant)
│   ├── SupportChat/      # شات خدمة العملاء للداخلية
│   ├── ChartsComponents/ # مكونات الإحصائيات والرسوم
│   ├── ProductsComponents/ # إدارة عرض وإضافة المنتجات
│   ├── LayoutComponents/ # مكونات الهيكل العام (Sidebar, Nav)
│   ├── LandingComponents/# مكونات الصفحة الرئيسية
│   └── globalComonents/  # مكونات عامة مثل (Loading, Buttons)
├── lib/              # المكتبات والخدمات المخصصة:
│   └── assistant/        # بيانات وخدمات المساعد الذكي (JSON & Voice)
├── css/              # التنسيقات العامة والخاصة بالصفحات
├── utils/            # وظائف مساعدة عامة (Helpers)
└── App.jsx           # الملف الرئيسي لإدارة المسارات (Router)
```

## 🛠️ التقنيات المستخدمة (Tech Stack)

*   **الواجهة الأمامية:** React (Vite)
*   **إدارة الحالة:** Redux Toolkit
*   **قواعد البيانات والمصادقة:** Supabase
*   **التصميم والتنسيق:** Vanilla CSS & Bootstrap (دعم RTL كامل)
*   **الذكاء الاصطناعي:** ElevenLabs API للخدمات الصوتية
*   **المكتبات الإضافية:** 
    *   Framer Motion (حركات تفاعلية)
    *   Recharts (إحصائيات)
    *   React Leaflet (خرائط)
    *   FontAwesome (أيقونات)

---

## ⚙️ معالجة البيانات (Project Logic)

1.  **المصادقة:** تعتمد على Supabase Auth مع تخزين الـ UserID في الـ Session.
2.  **الحماية:** استخدام `ProtectedDashboardRoute` لضمان عدم وصول غير المصرح لهم.
3.  **التدفق الوظيفي:** يتم سحب البيانات من Supabase وتوزيعها عبر Redux Store لتصل لكل المكونات بسلاسة.
